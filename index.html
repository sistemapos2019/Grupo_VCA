Sistema de ventas Lynx

Es un sistema que sirve para efectuar ordenes de los clientes en restaurantes o negocios que se dediquen a la venta de alimentos preparados.
Lynx es capaz de administrar las funciones que un negocio realiza como el manejo de mesas, meseros, inventarios, etc. Y es capaz de mostar estadisticas de lo vendido tanto diario como semanalmente.

Caracteristicas

  • Portabilidad

    Por su diseño web es facilmente acesible desde cualquier dispositivo ya sea escritorio o movil.

  • Utilidad

    Lynx ofrece una manera funcional y sencilla de manejar cualquier negocio dedicado a la venta de alimentos preparados o restaurantes.
    Propociona una manera sencilla para administrar mesas, meseros, platillos y mostrar resumenes tanto de ventas como contables.

  • Diseño

    Lynx ofrece una diseño totalmente responsive y amigable con el ususario con colores agradables.

Instalación

Como Lynx posee la licencia MIT puede ser instalado por cualquier persona que desee usarlo.

Para la instalacion de Lynx es necesario contar con el JDK de Java 8 instalado, NodeJS con su version 12 como minimo y un gestor de base de datos que puede ser tanto MariaDB o MySQL.
Para proceder a la instalacion deben seguirse los siguientes pasos:

 Repositorio de GitHub

INSTALACION NORMAL

  • Descargar el codigo fuente del repositorio de GitHub y usar la rama Master.

    •  BASE DE DATOS

  • Instalar un gestor de base de datos ya sea MariaDB o MySQL en sus versiones mas recientes.
  • Montar la base de datos con el script que se le provee en el repositorio de GitHub, para mas facilidad puede ayudarse de herramientas como MySQL WorkBench o PHPMyAdmin.

    •  BACKEND

  • Compilar el Backend usando mvn package -f /path/to/pom.xml.
  • Subir el .war a un servidor de aplicaciones.
  • Si su base de datos no esta ejecutandose en localhost y en el puerto 3306 tendra que entrar al codigo fuente y modificarlo por su direccion y puerto.

    •  FRONTEND

  • Si el backend no esta en localhost debera modificar el codigo fuente del Frontend y agregar la direccion de su servidor.
  • En la capeta donde se encuentra el Frontend ejecutar el comando "npm start" para compilar y ejecutar el Frontend.
  • Si hizo todo bien debera tener su sistema Lynx funcionando correctamente.

INSTALACION CON DOCKER

  • Descargar el codigo fuente del repositorio de GitHub y usar la rama Master.

    •  BASE DE DATOS

  • Descargar un contenedor de MariaDB con su ultima version, se recomienda la oficial.
  • Crear el contonedor de MariaDB.
  • Montar el script de la base puede usar herramientas como MySQL WorkBench o PHPMyAdmin.

    •  BACKEND

  • Compilar el Backend usando mvn package -f /path/to/pom.xml.
  • Descargar un contenedor de un servidor de aplicaciones, se recomienda Payara.
  • Ejecutar el contenedor del servidor de aplicaciones con el .war desplegado.
  • Si el contenedor no esta en localhost debera modificar el codigo fuente y agregar la direccion en la cual se encuentra su base de datos.

    •  FRONTEND

  • Si el contenedor del Backend no esta en localhost debera modificar el codigo fuente del Frontend con la direccion en la cual se encuentra el Backend.
  • Descargar un contenedor de NodeJS y ejecutar el Frontend en el.
  • Si hizo todo bien debera tener su sistema Lynx funcionando correctamente.
